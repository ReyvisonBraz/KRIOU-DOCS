# Etapa 06 — Componentes e design system

## Objetivo

Consolidar primitives, tokens e padrões de interação sem criar um framework interno excessivo.

## Tarefas

- inventário de componentes duplicados;
- tokens para cor, tipografia, espaço, raio, sombra e movimento;
- Button, Input, Select, Dialog, Toast, Card e Navigation com variantes;
- estados loading, empty, error, disabled e success;
- documentação visual e testes de interação;
- remover estilos inline repetidos por migração gradual;
- separar catálogo de ícones do componente de renderização.

## Critérios de aceite

- aparência consistente em todas as features;
- componentes possuem contrato pequeno e acessível;
- temas não dependem de duplicação de objetos de estilo;
- regressões visuais críticas possuem screenshots de referência.
